# 🏨 Restful Booker API Testing Project

This project demonstrates a comprehensive API Testing suite for the [Restful Booker API](https://restful-booker.herokuapp.com/). It includes a full testing lifecycle, from creating a Test Plan and writing manual Test Cases to automating the requests using **Postman** and running them via **Newman**.

## 📑 Project Overview
The goal of this project is to validate the CRUD (Create, Read, Update, Delete) operations of the Booker API, ensuring data integrity and correct status codes across different scenarios.

### 🛠 Tools Used
* **Postman:** For API request creation, environment management, and automation scripts (JavaScript).
* **Newman:** To run Postman collections via the Command Line Interface (CLI).
* **MS Excel / Word:** For Test Planning and Test Case documentation.
* **GitHub:** For version control and documentation.

---

## 📂 Repository Structure
* `BookerAPI_collection.json`: The exported Postman collection containing all API requests and test scripts.
* `BookerAPI_environment.json`: Environment variables (Base URL, Token, BookingID) for seamless execution.
* `Test Plan - Restful Booker ATB12x.docx`: Formal document outlining the testing strategy, scope, and resources.
* `Test cases - Ultimate.xlsx`: Detailed manual test cases including steps, expected results, and actual results.
* `test cases nuevo.xlsx`: Updated/New test scenarios.
* `/newman`: Folder containing HTML execution reports generated by Newman.

---

## 🧪 Scenarios Covered
1.  **Auth:** Create Token for access.
2.  **Booking:**
    * Get all Booking IDs.
    * Create a New Booking.
    * Get Booking details by ID.
    * Update Booking (Full Update).
    * Partial Update Booking (PATCH).
    * Delete Booking.
3.  **Health Check:** Verify API availability.

---

## 🚀 How to Run the Tests

### Option 1: Using Postman
1.  Import `BookerAPI_collection.json` into Postman.
2.  Import `BookerAPI_environment.json` into Postman.
3.  Select the **BookerAPI_environment** in the top right corner.
4.  Open the **Collection Runner** and click **Run BookerAPI**.

### Option 2: Using Newman (CLI)
Ensure you have Node.js and Newman installed:
```bash
npm install -g newman
npm install -g newman-reporter-htmlextra

Run the following command in your terminal:****

newman run BookerAPI_collection.json -e BookerAPI_environment.json -r htmlextra
The report will be generated in the newman/ folder.
